# 值得买 - 股票版

## 核心原则

把用户的交易纪律执行为趋势跟随与相对强度框架。它用于判断买入资格和持仓风险，不用于预测价格，也不下单。

- 默认只使用 Alpaca CLI / Alpaca Market Data API。不要使用其他券商连接器、网页搜索、新闻、社交媒体、分析师观点或模型记忆作为证据。
- 不创建、修改或取消订单。不要打印、保存或暴露 Alpaca 密钥。
- 必须输出明确结论和评分：`是否值得买`、`纪律评分 X/100`、触发的强制排除条件和一句话依据。
- 默认输出决策摘要，不输出思考过程、草稿、工具计划、逐步计算过程或长篇分析。
- 缺少关键数据时，不给买入结论；标记 `无法确认`，并说明被阻断的检查项。
- 优先选择趋势延续和相对强度领先标的；排除估值叙事、弱势修复、均值回归反弹和相对强度落后的标的。

## Alpaca 数据流程

默认快速路径只需要一次最新行情调用和一次历史日线调用；多标的时把所有 ticker 与 `SPY,QQQ` 一起批量请求。

1. 只在需要确认 CLI 存在时运行：

   ```bash
   alpaca version
   ```

   不默认运行 `alpaca doctor`；仅在数据调用失败、凭证或连接需要排障时使用。Trading API 失败不阻断本 skill 的市场数据分析。

2. 选择并标注 feed：

   ```bash
   FEED="${ALPACA_DATA_FEED:-iex}"
   ```

   用户明确要求或订阅支持时可用 `sip`。若 `sip` 返回 403/订阅限制，重试一次 `iex` 并在输出中标注限制。

3. 获取当日行情：

   ```bash
   alpaca data multi-snapshots --symbols {TICKER},SPY,QQQ --feed "$FEED" --quiet
   ```

   提取最新价、成交时间、bid/ask、价差、当日开盘/最高/最低/当前价、前收、日内区间位置、当日涨跌幅、跳空幅度、成交量和数据是否陈旧。只有批量 snapshot 缺字段时才使用单票 snapshot/latest-trade/latest-quote 作为备用调用。

4. 获取历史日线并计算全部量化指标——用本 skill 自带脚本，不要靠手算递归指标：

   ```bash
   python3 "$SKILL_DIR/scripts/indicators.py" \
     --symbols {TICKER},SPY,QQQ \
     --feed "$FEED" \
     --adjustment split
   ```

   `--start`/`--end` 可省略：默认 `--end` 为今天、`--start` 为约两年前（足够 MA60 与周线 MACD 预热）。仅在需要固定回测窗口时显式传日期。

   其中 `$SKILL_DIR` 为本 skill 目录。脚本内部调用 `alpaca data multi-bars`（约两年已完成日线，自动处理分页），仅用 Python 标准库确定性计算并输出 JSON：日线 MA10/20/30/60 及 MA60 方向、52 周高点距离、30 日结构（收益率/区间位置）、1/3/6 个月收益率、相对 SPY/QQQ 强度、MACD(12,26,9)、RSI(14/6)、KDJ(9,3,3)、量价（量能比），以及周线 MA5/10/20/30、周线 MACD 和周线空头排列判定。MACD/RSI/KDJ 这类递归指标必须以脚本输出为准，不得手算估计。脚本失败或字段缺失时，相应检查项标记 `无法确认`。

   底层日线接口（脚本内部使用，仅在排障或脚本不可用时手动调用）：

   ```bash
   alpaca data multi-bars --symbols {TICKER},SPY,QQQ \
     --start {YYYY-MM-DD} --end {YYYY-MM-DD} \
     --timeframe 1Day --adjustment split --feed "$FEED" --limit 10000 --quiet
   ```

5. 只在 ticker 有歧义或需要确认资产状态（交易所、是否可交易）时，懒加载资产元数据：

   ```bash
   alpaca asset get --symbol-or-asset-id {TICKER} --quiet
   ```

   如果该端点因仅有 Market Data 凭证而失败，价格和趋势检查可以继续。

## 技术指标计算与解读

数值一律取自 `scripts/indicators.py` 的输出（上一步已运行），脚本基于同一次日线调用、仅用已收盘日线确定性计算，不发起额外 Alpaca 调用、也不靠模型手算。下列公式是脚本实现的口径，用于解读字段、不用于重新心算。指标在本框架中是**趋势确认与时机工具，不是反转预测工具**：强势主升中 RSI/KDJ 长期高位属于强势特征，不能据此给卖出或回避；只有当指标与趋势/相对强度结构同时转弱时才下调评级。脚本输出字段对照：MACD → `macd.{DIF,DEA,hist,above_zero,bull,recent_cross}`；RSI → `rsi.{RSI14,RSI6}`；KDJ → `kdj.{K,D,J,bull,above_50}`；量价 → `volume.{ratio_vs_ma20,ratio_vs_ma50}`（口径为上一完整交易日全日量 vs 均量；盘中放量确认改用 snapshot 当日量）；周线 → `weekly.{MA5..MA30,macd,bearish_alignment,last_week_partial}`。

数据完整性字段：`ma.above_MA60`/`ma.MA60_rising` 为 `null` 表示历史不足 60 根、**MA60 无法计算**——此时风控条件 1 记 `无法确认`，**不得**当作"跌破 MA60"触发强制排除。`bars_count` 给出可用日线根数；`structure_30d.high_lookback_bars` < 252 时 52 周高点口径名不副实，相对强度/52 周距离需谨慎。`weekly.last_week_partial` 为 `true` 时最后一根周线尚未收盘，周线交叉/空头排列判定按未确认对待。

- **MACD(12, 26, 9)**：DIF = EMA12 − EMA26，DEA = DIF 的 EMA9，柱状值 = 2 ×(DIF − DEA)。关注：DIF 是否在 DEA 上方（多头）、柱状值是否放大还是收敛、DIF/DEA 是否在零轴上方、近 30 日是否出现金叉/死叉、价格创新高而 DIF 不创新高的顶背离。
- **RSI(14)**（必要时附 RSI6 看短期）：50 为多空分界，>50 偏多、<50 偏空；70+ 超买、30− 超卖，但趋势股可长期处于 50–80。关注 RSI 是否守住 50、是否出现价格新高 RSI 不新高的顶背离。
- **KDJ(9, 3, 3)**：先算 9 日 RSV，K = 前 K×2/3 + RSV×1/3，D = 前 D×2/3 + K×1/3，J = 3K − 2D。关注 K 与 D 的金叉/死叉、是否在 50 上方运行、J 值极端（>100 短期过热、<0 短期超卖）、高位死叉提示回调。
- **量价（Volume）**：当前成交量相对 20 日/50 日均量的量能比；近 30 日量价配合——突破关键位与主升是否放量、回调是否缩量为健康，上涨缩量背离或高位放量长阴/连续派发为警示；可选 OBV 看资金累积是否与价格同向。
- **周线确认（可选）**：日线信号矛盾或临界时，用周线序列复算周线 MACD / RSI 作为更高级别过滤，周线方向优先于日线噪音。

## 风控与入场条件

每次都按下列条件输出 `满足`、`不满足` 或 `无法确认`。

1. 日线 MA60 风险过滤：最新已完成日线收盘价低于 MA60 为不满足；最新价盘中跌破 MA60 需标记即时风险。若 `above_MA60` 为 `null`（历史不足 60 根），记 `无法确认`，不得当作不满足。
2. 周线趋势过滤：周线 MA5 <= MA10 <= MA20 <= MA30，或价格低于主要周均线且均线下行，为不满足。
3. 主趋势要求：清晰下行趋势、平台破位、突破失败、弱势反弹或均值回归反弹，不满足买入资格。
4. 相对强度要求：30 日、3 个月、6 个月表现应优于或至少不明显落后 SPY/QQQ。3 个月和 6 个月均跑输，或距离 52 周高点超过约 25% 且没有新的相对强度突破，为不满足。
5. 30 日结构确认：30 日收益率、区间位置、MA10/MA20/MA30 和高低点结构必须支持趋势延续。单日上涨不能抵消 30 日弱势结构。
6. 当日交易确认：当日行情应提供多头确认。冲高回落、接近日低、跌破关键位、价差异常或数据陈旧时，不提高买入评级。
7. 技术指标确认：MACD、RSI、KDJ 应多数与上行趋势一致——MACD 多头排列且在零轴上方、RSI 守住 50、KDJ 在 50 上方或刚金叉。出现 MACD/RSI 顶背离、高位死叉或多数指标同时转弱时，不提高买入评级。
8. 量价确认：突破与主升应有量能支持，回调缩量为健康。上涨缩量背离、高位放量派发或量能持续萎缩时，不提高买入评级。

强制排除条件：第 1-4 项任一 `不满足`，新买入结论必须为 `否`。第 5-8 项不满足时，即使前 4 项通过，也通常只能给 `观察`，除非用户明确询问的是已持仓风险管理。技术与量价指标仅作确认，不能单独推翻已通过的趋势与相对强度结论，也不能仅凭超买就给回避。

## 评分规则

始终给出 `X/100`，即使触发强制排除条件。触发强制排除条件时，分数仅作信息参考，不改变回避或减风险结论。

- 基础合规：20 分。可正常交易的美股标的（active/tradable）给满分。
- 绝对趋势：20 分。日线收盘价高于 MA60、MA60 上行、价格位于关键日/周均线之上、无周线空头排列。
- 相对强度：20 分。30 日、3 个月、6 个月相对 SPY/QQQ 领先，接近 52 周或阶段高点。
- 主升结构：15 分。高点/低点抬升、突破延续、30 日强势整理、无明显修正或破位结构。
- 技术指标（MACD/RSI/KDJ）：15 分。MACD 多头且在零轴上方、RSI 守住 50、KDJ 在 50 上方或金叉，多数指标与趋势一致；出现顶背离、高位死叉或多数转弱时扣分。
- 量价确认：5 分。突破与主升放量、回调缩量、量能与价格同向；上涨缩量背离或高位派发扣分。
- 当日与执行质量：5 分。当日多头确认、价差可接受、无明显冲高回落/流动性问题、风险位清晰。

评级映射：

- 85-100，强制排除条件为 `无`，且 30 日结构与当日确认通过：`是否值得买: 是`。
- 70-84，或 30 日/当日确认不足：`是否值得买: 观察`。
- 低于 70，或任一强制排除条件触发：`是否值得买: 否`。
- 已持仓且日线 MA60 失守或周线空头排列：`是否值得买: 持仓需减风险`。
- 数据不足以计算核心条件：`是否值得买: 无法评分`。

## 输出格式

默认中文，先结论后证据。不要输出思考过程或工具过程。

**结论**

- 是否值得买: `是`, `否`, `观察`, `持仓需减风险`, 或 `无法评分`
- 建议: `可关注/小仓位试探`, `观察等待`, `回避`, `减仓/退出`, 或 `无法评分`
- 纪律评分: `X/100`
- 强制排除条件: 列出触发项；没有则写 `无`
- 一句话: 最关键决策依据

**关键证据**

只列 4-7 条：

- 日线趋势：最新完成日线收盘价相对 MA60，最新价相对 MA60
- 周线趋势：周线 MA5/MA10/MA20/MA30 排列
- 30 日结构：30 日收益率、区间位置、MA10/MA20/MA30、结构分类
- 技术指标：MACD（DIF/DEA/柱状、零轴）、RSI(14)、KDJ(9,3,3) 及是否有背离/金叉死叉
- 量价确认：当前量能比、近 30 日量价配合、是否放量突破或缩量背离
- 当日确认：当日涨跌幅、日内区间位置、买卖价差、确认分类
- 相对强度：30 日、3 个月、6 个月相对 SPY/QQQ 表现

**风控过滤条件**

表格列名固定为：`过滤条件`, `状态`, `关键证据`, `处理建议`。

**评分拆解**

列出七个评分项及得分（基础合规、绝对趋势、相对强度、主升结构、技术指标、量价确认、当日与执行质量）。触发强制排除条件时说明分数仅供参考。

**建议**

给出一个明确动作：新开仓、继续观察、回避、减仓/退出，或补充数据后重评。

只有用户要求详细数据时，才追加 **Alpaca 数据明细**，包括标的名称、交易所、状态、feed、复权方式、备用调用、最新成交/报价、日线/周线/30 日/当日字段、基准收益对比和订阅限制。
