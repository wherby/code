
# 怎么维护下凸包
后半部分用凸包的时候，为什么(k1 - k) * (b2 - b) >= (k2 - k) * (b1 - b): lines.pop() 这个时候就能排除上一个点？

这就是维护**下凸壳（Lower Convex Hull）**时的核心几何判定逻辑，通常被称为 **"斜率优化"** 或 **"三点共序检查"**。

当我们将直线方程写成 $y = kx + b$ 的形式时，要维护一个下凸壳（即在任何 $x$ 处都取最小值的直线集合），我们需要保证这些直线的**斜率 $k$ 是单调递减（或递增）的**，并且相邻两直线的**交点横坐标也是单调的**。

### 1. 直观理解：为什么要剔除？
想象你有三条直线 $L_1, L_2, L_3$，它们的斜率依次减小（$k_1 > k_2 > k_3$）。
如果在 $L_1$ 和 $L_2$ 还没来得及交汇之前，$L_3$ 就已经跑到了 $L_2$ 的下面，那么 $L_2$ 就变成了“多余”的直线——也就是说，**在任何 $x$ 位置，$L_2$ 都不是那个最小的值。**



### 2. 数学推导：交点判定
我们要比较的是：
* $L_1$ 和 $L_2$ 的交点横坐标 $x_{12}$
* $L_2$ 和 $L_3$ 的交点横坐标 $x_{23}$

对于直线 $y = kx + b$，两条直线 $L_i, L_j$ 的交点为：
$$k_i x + b_i = k_j x + b_j \implies x_{ij} = \frac{b_j - b_i}{k_i - k_j}$$

在下凸壳中，为了让 $L_2$ 有存在的价值，必须满足 $x_{12} < x_{23}$。
如果 **$x_{12} \ge x_{23}$**，说明 $L_2$ 被 $L_1$ 和 $L_3$ 给“覆盖”掉了，$L_2$ 永远无法成为最小值，必须弹出（pop）。

代入公式：
$$\frac{b_2 - b_1}{k_1 - k_2} \ge \frac{b_3 - b_2}{k_2 - k_3}$$

### 3. 为什么代码里是乘法？
直接用除法会有两个问题：**浮点数精度误差** 和 **被零除的风险**。
因此，我们通过交叉相乘将分式转换为整式：
$$(b_2 - b_1)(k_2 - k_3) \ge (b_3 - b_2)(k_1 - k_2)$$

如果你代码中的变量命名稍有不同（比如 $k$ 和 $b$ 分别代表当前准备加入的直线 $L_3$，$k_1, b_1$ 是栈顶往下一个，$k_2, b_2$ 是栈顶），那么这个公式写出来就是你看到的那个样子：
$$(k_1 - k) \cdot (b_2 - b) \ge (k_2 - k) \cdot (b_1 - b)$$



### 4. 几何意义
这个不等式本质上是在比较**向量的叉积**，或者说是在检查**斜率的变化率**：
* 左边和右边分别代表了两对直线交点的相对位置。
* 如果条件成立，说明 $L_2$ 形成的“拐角”是向外突出的（上凸），而在维护下凸壳时我们只想要向内凹的形状。

### 总结
当你发现新加入的直线 $L$ 使得原本的栈顶直线 $L_{top}$ 变得毫无意义（即 $L_{top}$ 对应的区间已经缩减为零甚至负数时），`lines.pop()` 就能确保你的凸壳始终由**最优**的直线组成。这样在查询时，你只需要在这一组单调的直线中进行二分即可。
