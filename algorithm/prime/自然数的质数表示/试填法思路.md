当 $N$ 很大（例如 $10^{18}$），而我们无法显式构造出整个 $O(N)$ 序列时，**“试填法”（Digit DP-like Greedy Construction / Lexicographical Counting）** 是解决“查找第 $K$ 个”问题的标准暴力美学。

其核心思想是：**按位确定答案。对于当前的每一个决策分支，计算该分支下包含多少个合法的数。如果 $K$ 大于这个计数，就跳过该分支并减去计数；否则，进入该分支。**

为了实现这个过程，我们需要一个核心工具：**计算在给定前缀下，有多少个数的质因数序列符合字典序。**

---

### 1. 问题的转化：质因数个数统计

假设我们已经确定的数是 $V$（即前缀），当前最小可选的质数是 $P_i$。
我们需要计算满足以下条件的整数 $X$ 的个数：
1. $X = V \times p_{j_1} \times p_{j_2} \times \dots$
2. $P_i \le p_{j_1} \le p_{j_2} \le \dots$
3. $X \le N$

这个问题等价于：**在剩下可用的质数 $\{P_i, P_{i+1}, \dots\}$ 中，有多少种选取方式（可重复）使得乘积 $\le \lfloor N/V \rfloor$？**



### 2. 试填法步骤

我们将这个过程想象成在字典序树上移动：

1.  **确定当前数**：从 $V=1$ 开始。$1$ 总是占据第 1 位。如果 $K=1$，返回 $1$。
2.  **枚举质数分支**：
    从最小的可用质数 $P_i$ 开始枚举：
    * 计算分支 $P_i$ 的**总贡献量**。这个分支包含：
        * 数 $V \times P_i$ 本身。
        * 所有以 $V \times P_i$ 为前缀，且后续质因数 $\ge P_i$ 的数。
    * **判断 $K$ 的落点**：
        * 如果 $K \le \text{Count}(V \times P_i, P_i)$：说明目标数就在这个分支里。令 $V = V \times P_i$，$K = K - 1$（扣掉 $V \times P_i$ 自身），继续在该分支下试填。
        * 如果 $K > \text{Count}(V \times P_i, P_i)$：说明目标数不在这个分支。令 $K = K - \text{Count}(V \times P_i, P_i)$，尝试下一个质数 $P_{i+1}$。

### 3. 如何高效计算 $\text{Count}(Limit, \text{min\_prime})$？

这是问题的难点。当 $N$ 很大时，这是一个类似**数论函数求和**或**背包 DP** 的问题。

* **动态规划/记忆化搜索**：
    定义 $G(m, i)$ 为使用不小于 $P_i$ 的质数，能乘出的 $\le m$ 的数的个数。
    $$G(m, i) = 1 + \sum_{j=i}^{\pi(m)} G(\lfloor m/P_j \rfloor, j)$$
    *其中 $1$ 代表数 $1$ 本身。*

* **Lucy Hedgehogs / Min_25 筛思想**：
    如果 $N$ 达到 $10^{11}$ 以上，简单的 DFS 计数会太慢。我们需要利用类似 Min_25 筛的方法，将质数分为“小质数”和“大质数”处理。
    * 对于大质数（$P_j > \sqrt{m}$），它们在乘积中只能出现一次。
    * 此时 $\sum G(\lfloor m/P_j \rfloor, j)$ 简化为统计区间 $[P_i, m]$ 内质数的个数 $\pi(m) - \pi(P_i - 1)$。

### 4. 伪代码实现思路

```python
def count_combinations(limit, prime_start_idx):
    # 如果 limit 很小，直接返回 1
    if limit < primes[prime_start_idx]:
        return 1
    
    # 记忆化...
    res = 1 # 代表 limit 自身这个前缀
    for i in range(prime_start_idx, len(primes)):
        p = primes[i]
        if p * p > limit:
            # 剩余的数只能是单个大质数，使用预处理的质数计数函数 pi(x)
            res += max(0, pi_function(limit) - pi_function(p - 1))
            break
        
        # 递归计算更高幂次或后续质数
        res += count_combinations(limit // p, i)
    return res

def find_kth(N, K):
    current_v = 1
    if K == 1: return 1
    K -= 1 # 扣除 1
    
    p_idx = 0
    while K > 0:
        p = primes[p_idx]
        if current_v * p > N: break # 理论上不应发生
        
        cnt = count_combinations(N // (current_v * p), p_idx)
        if K <= cnt:
            # 目标在当前质数分支
            current_v *= p
            K -= 1
            if K == 0: return current_v
            # 下一轮继续从 p_idx 开始（因为可以重复乘同一个质数）
        else:
            # 跳过当前质数分支
            K -= cnt
            p_idx += 1
    return current_v
```

### 总结
当 $N$ 巨大时，这题变成了**“在质因数分解空间内的字典序搜索”**。

* **小 $N$ ($< 10^7$)**：直接 $O(N)$ 构造（如你写的代码）。
* **中 $N$ ($10^7 \sim 10^{11}$)**：试填法 + 基础记忆化搜索。
* **大 $N$ ($> 10^{12}$)**：试填法 + 结合 **Meissel-Lehmer** 或 **Min_25 筛** 快速计算 $\pi(x)$ 来加速计数。
