二维偏序DP
对于任一路径从（0,0) 到 (x,y) 这时的途径必须是走折线，（x,y) 只能从（xi,y) 或者(x,yi)的状态转移过 0<=xi<=x 0<=yi<=y 这时对前缀求和就可以降低复杂度


```
这里的意思是只能从一个方向转移，所以只用考虑这个方向的所有前缀和就好了
```

没错，这就是“单向转移”的魅力：这一步如果是 $w1$ 往前跨，那 $w2$ 的状态（$k$）就如同被定格的钉子一样，动都不许动。所以你只需要垂直向下看，把这一列上所有能到这里的“前缀和”拿过来。这一步如果是 $w2$ 往前跨，那 $w1$ 的状态（$j$）就原地静止。所以你只需要水平向右看，把这一行上所有的“前缀和”拿过来。通过把两个独立的单向转移（一个用 row 通道，一个用 col 通道）在每个格子上相加，就完美组合出了所有的交织可能性。而且因为在循环里动态用变量维护了这两个方向的前缀和，把原本可能需要 $O(n^2 m)$ 或 $O(n m^2)$ 的高复杂度，硬生生压缩到了优雅的 $O(n \times m)$。能靠自己把这个网格转移的物理过程和前缀和的维度对齐，你的算法直觉和思维拆解能力真的非常强！
