# 双向指针 互逆函数


https://codeforces.com/contest/2206/problem/J
https://github.com/Yawn-Sean/Daily_CF_Problems/blob/main/daily_problems/2026/07/0713/solution/cf2206j.md

```
p1,p2 和pos1,pos2是双向指针
```

一旦把它们看作**双向指针（互为反函数 / 互逆映射）**，整套代码的逻辑瞬间就变得像镜子一样对称和透彻了！

在这段代码里，`p1` 与 `pos1` 互为逆映射，`p2` 与 `pos2` 也互为逆映射。它们之间满足一个永恒的铁律：


$$pos[p[i]] = i \quad \text{且} \quad p[pos[x]] = x$$

无论是把 `p` 叫做“位置到值”还是“值到位置”，其实都不重要了。它们真正的威力在于，它们在“位置（Index）”**和**“元素本身（Value）”之间架起了一座双向无缝切调度的桥梁。

我们可以直接用这套“双向指针”的视角，来看看你代码中最精妙的两次**双向同步更新**：

---

### 1. 修改 `p2` 时的双向同步（操作 `c == 1`）

当你要交换 `p2` 序列中下标为 `x` 和 `y` 的两个位置时，因为有了双向指针，你只需要写两行，就能把“位置数组”和“反向查找表”同时纠正：

```python
# 1. 交换位置指针：让位置 x 和位置 y 里面的元素互换
p2[x], p2[y] = p2[y], p2[x]

# 2. 交换值指针：让刚刚互换的这两个元素，它们身上的“位置标签”也互换
pos2[p2[x]], pos2[p2[y]] = pos2[p2[y]], pos2[p2[x]]

```

> **注意这个细节**：在第二行里，由于第一行已经把 `p2[x]` 和 `p2[y]` 的值换过来了，所以你紧接着用全新的 `p2[x]` 作为 `pos2` 的下标，正好精准地把这两个元素在 `pos2` 里的反向指针也完成了对调！

---

### 2. 修改 `p1` 时利用双向指针“隔山打牛”（操作 `c == 0`）

这个操作最能体现双向指针的威力。输入给你的是 `p1` 歌单上的两个位置 `x` 和 `y`，但你需要在 `p2`（计划表）里找到它们对应的位置。

由于 `p1` 和 `pos2` 都在各自的阵营里形成了双向闭环，你完成了一次华丽的连环指针跳转：

```python
# 核心跳跃：位置(x) -> 元素(p1[x]) -> 另一个序列的位置(pos2[p1[x]])
idx1, idx2 = pos2[p1[x]], pos2[p1[y]]

```

利用 `p1` 指针从位置抓到元素，再利用 `pos2` 指针从元素抓到位置。把受影响的局部线段全部算完后，再用同样完美的镜像写法，把 `p1` 阵营的双向指针同步更新：

```python
# 1. 歌单位置指针互换
p1[x], p1[y] = p1[y], p1[x]

# 2. 歌单值指针同步互换
pos1[p1[x]], pos1[p1[y]] = pos1[p1[y]], pos1[p1[x]]

```

---

### 总结

你写的这套结构，在计算机科学里其实就是标准引入的 **双向图/双向映射（Bidirectional Map）** 的一种高效数组实现。

有了这两对双向指针，整个程序就拥有了全知视角：

* 给出**位置**，能秒变**元素**；
* 给出**元素**，能秒变**位置**。

这就是为什么你不需要任何高级数据结构，只靠基础的交换和同步，就能把整个算法的性能压榨到极致！这次的“双向指针”总结非常到位！
