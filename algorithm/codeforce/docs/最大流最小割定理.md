这是一个非常经典的网络流（Maximum Flow）**问题，但由于其结构具有非常特殊的对称和串联特性，我们完全不需要写复杂的网络流算法（如 Dinic 或 ISAP），只需要用极其简洁的**最小割（Min-Cut）公式就能秒杀它。

### 1. 核心观察：串联分段的“木桶效应”

题目指出，整个通道是由多个完全相同的结构（Segments）**首尾相连**串成的一长条。

* 每一个分段的出口，就是下一个分段的入口。
* 所有的绵羊都必须经过这些连接点（Junctions）。

根据网络流的性质，整个系统的最大流（最多能逃脱的绵羊数量）受限于这串分段中**能力最弱（最大流最小）的那一个分段**。也就是说：

$$\text{Total Max Flow} = \min_{i=1}^{N} (\text{Max Flow of Segment } i)$$

---

### 2. 单个分段的最大流计算

我们把目光聚焦到任意一个单独的分段 $i$。它由 4 个顶点（入口 $S$、上顶点 $T$、下顶点 $B$、出口 $E$）和 5 条带有容量限制的路径组成：

* $A_i$: $S \to T$
* $D_i$: $S \to B$
* $C_i$: 连接 $T$ 和 $B$（根据题意，方向可以是上到下，也可以是下到上，等价于无向边）
* $B_i$: $T \to E$
* $E_i$: $B \to E$

根据 **最大流最小割定理（Max-Flow Min-Cut Theorem）**，要让 $S$ 和 $E$ 完全断开，所有可能的切断方案（割）的容量极小值就是该分段的最大流。在这个简单的钻石结构中，只有以下 4 种本质不同的割：

1. **直接切断入口：** 割掉 $A_i$ 和 $D_i$，容量为：$A_i + D_i$
2. **直接切断出口：** 割掉 $B_i$ 和 $E_i$，容量为：$B_i + E_i$
3. **左上到右下的斜对角割：** 割掉 $A_i$、$C_i$ 和 $E_i$（阻断了 $S \to T$ 且利用 $C_i$ 绕路到下面的可能），容量为：$A_i + C_i + E_i$
4. **左下到右上的斜对角割：** 割掉 $D_i$、$C_i$ 和 $B_i$，容量为：$D_i + C_i + B_i$

因此，单个分段 $i$ 的最大流量为：


$$\text{Max Flow}_i = \min(A_i + D_i, \; B_i + E_i, \; A_i + C_i + E_i, \; D_i + C_i + B_i)$$

---

### 3. 解题算法与伪代码

你只需要在输入数据时，对每个分段算一次上面的 $\min$，然后维护一个全局的最小值即可。

```python
def solve():
    # 假设输入包含 N 个分段，以及每个分段的 A, B, C, D, E 五条边的容量
    N = int(input())
    
    global_max_sheep = float('inf')
    
    for _ in range(N):
        A, B, C, D, E = map(int, input().split())
        
        # 计算当前分段的最小割（即最大流）
        current_segment_flow = min(
            A + D,
            B + E,
            A + C + E,
            D + C + B
        )
        
        # 全局木桶效应，取所有分段的最小值
        global_max_sheep = min(global_max_sheep, current_segment_flow)
        
    print(global_max_sheep)

```

**复杂度分析：**

* **时间复杂度：** $O(N)$。我们只需要对每个分段进行常数次加法和比较。
* **空间复杂度：** $O(1)$。可以边读入边计算，不需要存储庞大的图结构。
