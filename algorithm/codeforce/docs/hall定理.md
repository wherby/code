通过这套逻辑，你以后看到类似的“人与物品互相匹配，且有局部数量限制”的题目，都可以瞬间看破本质。

---

## 第一部分：用 Hall 定理解决此类问题的通用思考逻辑

这一类问题的核心矛盾通常是：**“每个人有总额度限制，而每个物品也有被选择的次数限制，问某种方案是否可行（或极值是多少）”**。

通用的思考逻辑分为以下 **4 个标准步骤**：

```
1. 建立二分图模型 (人 ↔ 物品)
       ↓
2. 转换为必要条件 (选物品子集 S，看人的最大承载力)
       ↓
3. 贪心优化 (对物品排序，只考察前缀子集，使限制最紧紧确)
       ↓
4. 代数变形/对偶转化 (分离变量，化简解出目标答案)

```

### 1. 建立二分图模型

把参与者放在左边（度数为 $R_j$），把物品/题目放在右边（度数为 $k_i$）。两边连边的容量通常是 $1$（即一个人对一个物品只能选一次）。

### 2. 导出必要条件（容量放缩）

任意选择右边物品的一个子集 $S$（大小为 $i$）。

* **左边的现实：** 这 $i$ 个物品在真实数据里，总共被选择了 $\sum_{j \in S} k_j$ 次。
* **右边的上限：** 假设所有人为了满足这个子集，都把潜力发挥到极限。每个人在这 $i$ 个物品里，最多能贡献 $\min(\text{自身总额度}, i)$ 次。
* **核心不等式：** $\text{真实发生数} \le \sum \min(\text{每个人的额度}, i)$。如果这个不等式在某个子集上被无情打破，说明全局方案根本无法建立。

### 3. 贪心排序，寻找“最紧限制”

子集有 $2^p$ 个，不可能穷举。但我们要找的是**最容易让不等式崩盘的瓶颈**。
所以把物品按度数**从大到小（或从小到大）排序**。只去考察前缀子集（前 $i$ 个物品），这时的真实发生数最大，限制最紧。

### 4. 代数变形与分离变量

利用排好序的前缀和，把含有未知数（比如你想求的人数 $v$ 或 $ans$）的项通过移项、提取公因式隔离出来，最终化简成一个类似 `ans <= 分子 // 分母` 的形式，取交集即可。

## 第二部分：本题的终极复盘与深度分析

回到这道题目，我们来看那个绝妙的 Python 代码是如何一步步践行这个逻辑，并完成神级优化的：

### 1. 巧妙的“对偶（补集）转化”

原问题看的是“做出的题数”，公式里会带有讨厌的 $\min(x, i)$，需要分类讨论。
代码的高明之处在于**视角切换**：

* 每个人总共没做的题目数（空位数）不能超过 $x$。
* 超过你的人（超人），总空位数最多只有 $x - 1$。
* 没超过你的人（普通人），总空位数最多有 $p$（相当于没有总数限制）。

### 2. 分析题目子集的保底限制

假设我们正在考察前 $i+1$ 道最热门的题（即空位最少的题），真实空位总数是 `cur`。

我们去压榨一个普通人，看他在眼前这 $i+1$ 道题里**最少最少也必须留下多少个空位**？
他总共要留 $x$ 个空位。为了在前面留得少，他把空位拼命往后面仍。但后面只有 $p - i - 1$ 道题，全扔满也只能扔掉 $p - i - 1$ 个。
所以，他**在眼前这几道题里保底必须留下：$x - (p - i - 1)$ 个空位**。

### 3. 建立容积不等式

* 每一个普通人，在眼前这几道题里，由于实力不行，**保底要贡献** $x - (p - i - 1)$ 个空位。
* 每一个超人，由于实力很强，他们在眼前这几道题里**保底只需要贡献 $0$ 个空位**。

因此，如果眼前一共有 `ans` 个普通人，他们能承担的**保底空位总量**就是：


$$\text{总保底空位数} = ans \times [x - (p - i - 1)]$$

这个由普通人实力不行而**被迫产生**的保底空位数，绝对不能超过眼前这几道题实际拥有的空位总数 `cur`（要是比 `cur` 还大，说明这群普通人就算把后面的题全部做不出来，眼前的空位也根本不够他们留的，这就矛盾了）。

所以得到极其干净的不等式：


$$ans \times [x - (p - i - 1)] \le cur$$

### 4. 代码的完美收官

由此移项，直接解出普通人人数 `ans` 的上限：


$$ans \le \frac{cur}{x - (p - i - 1)}$$

因为每一道题（每一个前缀 $i$）都会对普通人的人数上限提出要求，所以代码用 `fmin` 一路取交集，卡出普通人的**全局最大可能数量**。

最后，用总人数 $n$ 减去普通人的最大数量，剩下的自然就是**严格超过你的人数（超人）的最小值**。

### 核心结语

利用 Hall 定理解决这类问题的真谛，就是**在宏观总量守恒的大背景下，通过局部的最极端假设，将无序的分配问题，收敛为边界的容量卡口。** 这段代码通过“数空位”的对偶操作，把本需要分类讨论的二部图流问题，变成了简单优美的一维线性扫描，可以说是算法与数学结合的典范。
