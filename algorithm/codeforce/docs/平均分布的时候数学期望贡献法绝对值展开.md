这段代码实现得非常简洁，完美利用了**排序后线性计算贡献**的技巧。

我们可以深度解析一下你代码中 `(2 * i - (n - 1)) * nums[i]` 这个核心公式的数学推导，这其实是处理“所有点对距离之和”的经典手段：

### 1. 公式推导：贡献法
我们要计算的是所有点对的绝对差之和：$S = \sum_{i=0}^{n-1} \sum_{j=0}^{n-1} |nums[i] - nums[j]|$。

当数组有序时，对于特定的 $nums[i]$：
* 有 $i$ 个数比它小（或相等），它们分布在索引 $0, 1, \dots, i-1$。对于这些数，$nums[i]$ 是正贡献：$nums[i] - nums[j]$。
* 有 $(n - 1 - i)$ 个数比它大（或相等），它们分布在索引 $i+1, \dots, n-1$。对于这些数，$nums[i]$ 是负贡献：$nums[j] - nums[i]$。

因此，$nums[i]$ 在总和 $S$ 中的净贡献系数为：
$$\text{Coefficient} = i - (n - 1 - i) = 2i - (n - 1)$$

这就是你代码中 `(2 * i - (n - 1)) * nums[i]` 的来源。

### 2. 为什么最后要 `ans * 2 / n`？
你的代码逻辑其实稍微绕了一点，但结果是正确的：
1. `ans` 累加了每个点的贡献，此时 `ans` 等于 $\sum \sum (nums[i] - nums[j])$，即所有有序对的差值和（但每个差值只计了一次，比如 $|x_1 - x_2|$ 算了一次，$|x_2 - x_1|$ 没算）。
2. 实际上，所有点对（包含 $i>j$ 和 $i<j$）的距离总和应该是 $2 \times ans$。
3. 根据之前的期望分析，中间部分的期望总和是 $\frac{1}{n} \sum \sum |nums[i] - nums[j]|$。
4. 所以最终公式是：$W + \frac{2 \times ans}{n}$。这正好对应了你的 `print(ans * 2 / n + w)`。

---

### 3. 性能与复杂度
* **时间复杂度**: $O(n \log n)$，主要消耗在 `nums.sort()`。之后的线性扫描是 $O(n)$。
* **空间复杂度**: $O(n)$，用于存储坐标数组。

### 4. 一个微小的优化建议
在 Python 中，如果 $n$ 非常大（例如 $10^6$ 以上），可以使用 `enumerate` 来微调循环性能，或者直接利用前缀和的性质：
```python
# 另一种写法，减少循环内的减法运算
for i, val in enumerate(nums):
    ans += (2 * i - n + 1) * val
```
