

在圆环距离问题中，奇偶性的核心差异在于：**是否存在一个唯一的“最远点”（对跖点）**。

你的代码中 `v = (m + 1) // 2` 配合两个单调区间的差分处理，实际上通过“分段函数覆盖”巧妙地兼容了奇偶性。

### 1. 奇数情况：$m = 2k + 1$
当 $m$ 为奇数时，$v = (2k + 1 + 1) // 2 = k + 1$。
对于位置 $x$ 的人：
* **右半段**：距离从 $0$ 增加到 $k$（区间 $[x, x+k]$，长度为 $k+1$）。
* **左半段**：距离从 $k$ 减小到 $1$（区间 $[x+k+1, x+2k]$，长度为 $k$）。
* **总计**：$(k+1) + k = 2k+1 = m$ 个位置。

每个点都有对称的距离分布，逻辑非常完美。



---

### 2. 偶数情况：$m = 2k$
当 $m$ 为偶数时，$v = (2k + 1) // 2 = k$。
对于位置 $x$ 的人：
* **右半段**：距离从 $0$ 增加到 $k-1$（区间 $[x, x+k-1]$，长度为 $k$）。
* **左半段**：距离从 $k$ 减小到 $1$（区间 $[x+k, x+2k-1]$，长度为 $k$）。
* **总计**：$k + k = 2k = m$ 个位置。

**这里的关键点在于：**
在偶数长度的环中，距离 $x$ 最远的点只有一个，即 $x+k$，其距离为 $k$。
在你的代码里：
* 第一段（右侧）更新到 $x+v-1$（即 $x+k-1$），最大距离是 $(x+k-1) - x = k-1$。
* 第二段（左侧）从 $x+v$（即 $x+k$）开始，斜率为 $-1$，常数项为 $x+m$。
* 代入 $i = x+k$：距离 $= (x+2k) - (x+k) = k$。

所以，当 $m$ 为偶数时，最远点 $k$ 被归到了“左半段”的起点。



---

### 3. 为什么 `v = (m + 1) // 2` 是通用的？

我们可以列一张表看看 $v$ 划分的区间终点：

| $m$ | $v$ | 右半段区间 $[x, x+v-1]$ | 左半段区间 $[x+v, x+m-1]$ | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **5 (奇)** | 3 | $[x, x+2]$ (距离 0,1,2) | $[x+3, x+4]$ (距离 2,1) | 左右完全对称 |
| **6 (偶)** | 3 | $[x, x+2]$ (距离 0,1,2) | $[x+3, x+5]$ (距离 3,2,1) | 最远距离 3 被划入左侧 |

**结论：**
这种写法本质上是把圆环拆成了两个“爬坡”的一次函数。
* 如果 $m$ 是奇数，波峰（最远距离）在两段交界处平滑过渡。
* 如果 $m$ 是偶数，波峰刚好落在第二段的开头。

### 4. 潜在的微小偏差
如果题目要求 $m$ 为偶数时，最远点 $m/2$ 必须严格对称处理（比如某些特殊权重的题目），目前的写法会将最远点视为“下降沿”的起点。但在计算**总和最小值**时，这种 $O(1)$ 的区间偏移不会改变最终 `min(ans)` 的结果，因为所有 $x$ 都遵循同样的偏移逻辑。
