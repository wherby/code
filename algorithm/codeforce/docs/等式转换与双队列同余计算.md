# [把求的等式转换为两个数列的差，然后用双数列同余的方式计算](../数论/等式转换与双数量同余计算.py)


这也是一道非常经典的**数论与前缀和/贡献转换**结合的题目。通过将看似复杂的“两两乘积之和”进行代数变形，我们可以将区间查询转化为简单的前缀和余数统计，从而在 $O(n)$ 的纯线性时间内完美解决。

下面我们一步步来推导和拆解这道题。

---

## 1. 核心突破口：两两乘积和的代数变形

题目定义一个区间的“权重”为所有两两乘积之和：


$$\text{Weight} = \sum_{1 \le i < j \le m} b_i b_j$$

在初中数学中，我们学过一个非常经典的完全平方公式扩展：


$$(b_1 + b_2 + \dots + b_m)^2 = \sum_{i=1}^m b_i^2 + 2 \sum_{1 \le i < j \le m} b_i b_j$$

移项之后，我们就能把两两乘积之和（Weight）写成更易于处理的形式：


$$2 \times \text{Weight} = (b_1 + b_2 + \dots + b_m)^2 - \sum_{i=1}^m b_i^2$$

由于题目只关心权重能否被 **3** 整除，也就是求 $\text{Weight} \equiv 0 \pmod 3$。
因为 2 和 3 互质，所以在模 3 的世界里，乘以 2 不会影响整除性。公式在模 3 意义下等价于：


$$(b_1 + b_2 + \dots + b_m)^2 - \sum_{i=1}^m b_i^2 \equiv 0 \pmod 3$$


也就是：


$$(\text{区间元素和})^2 \equiv \sum \text{区间元素的平方} \pmod 3$$

---

## 2. 模 3 的乘法特征简化

在模 3 的世界里，任何整数的平方都有非常美妙的性质：

* 若 $x \equiv 0 \pmod 3$，则 $x^2 \equiv 0 \pmod 3$
* 若 $x \equiv 1 \pmod 3$，则 $x^2 \equiv 1 \pmod 3$
* 若 $x \equiv 2 \pmod 3$，则 $x^2 \equiv 4 \equiv 1 \pmod 3$

这意味着：**任何数的平方在模 3 下要么是 0（当它能被3整除），要么是 1（当它不能被3整除）！**
因此：

* $\sum \text{区间元素的平方} \pmod 3$ 的物理意义，其实就是**区间内不能被 3 整除的元素的个数**。

---

## 3. 引入前缀和转化区间

设原数组为 $a$。为了快速求出任意区间 $[l, r]$ 的“元素和”与“非3倍数个数”，我们引入两个前缀和数组：

1. $S[k] = \sum_{i=1}^k a_i \pmod 3$：表示前缀元素和模 3 的余数。
2. $C[k] = \sum_{i=1}^k (a_i^2 \pmod 3) = \text{前缀中非3倍数的元素个数}$。

那么对于任意 subarray $[l, r]$，我们有：

* 区间元素和 $\equiv S[r] - S[l-1] \pmod 3$
* 区间元素平方和 $\equiv C[r] - C[l-1]$

代入我们第一步推导的判定核心方程：


$$(S[r] - S[l-1])^2 \equiv C[r] - C[l-1] \pmod 3$$

---

## 4. 状态归类与线性统计 ($O(n)$ 计数)

现在我们的目标是：寻找有多少对 $(l-1, r)$ 满足上述方程。如果直接枚举 $l$ 和 $r$，复杂度是 $O(n^2)$。但注意到方程左边只有 $S$ 的差的平方，右边是 $C$ 的差。

我们可以把当前位置 $r$ 的所有状态，用它之前的历史状态来进行归类统计。
对于一个固定的右端点 $r$，我们需要知道前面有多少个位置 $j = l-1$ 满足：


$$(S[r] - S[j])^2 \equiv C[r] - C[j] \pmod 3$$


移项整理，将包含 $r$ 的放左边，包含 $j$ 的放右边：


$$C[r] - (S[r] - S[j])^2 \equiv C[j] \pmod 3$$

由于 $S[r]$ 和 $S[j]$ 的取值只能是 $0, 1, 2$，我们可以直接根据 $S[j]$ 的可能取值分情况讨论：

### 状态定义：

在遍历的过程中，我们用一个动态计数器 `cnt[s][c_mod]` 来记录：**在当前位置之前，所有满足 $S[j] = s$ 且 $C[j] \equiv c\_mod \pmod 3$ 的历史位置 $j$ 的数量。**
因为 $s \in \{0,1,2\}$ 且 $c\_mod \in \{0,1,2\}$，这个计数器的大小只有固定的 $3 \times 3 = 9$。

当右端点推进到 $r$ 时，我们已知当前的 $S[r]$ 和 $C[r]$。我们去穷举历史可能存在的 $S[j]$ 的值（即 $s = 0, 1, 2$）：

1. 算出来左边的值：$target = (C[r] - (S[r] - s)^2) \pmod 3$
2. 那么，满足当前配对条件的历史位置个数就是 `cnt[s][target]`。
3. 把这个个数累加到最终答案中。
4. 最后，把当前位置自身的状态 $(S[r], C[r] \pmod 3)$ 作为一个新的历史记录加入到 `cnt` 中。

---

## 5. 算法实现 (Python)

```python
def solve_weight_divisible_by_3(n, a):
    # cnt[s][c] 表示历史前缀中：
    # 元素和模3为 s，且非3倍数个数模3为 c 的前缀个数
    cnt = [[0] * 3 for _ in range(3)]
    
    # 初始状态：l-1 = 0 的时刻，S[0] = 0, C[0] = 0
    cnt[0][0] = 1
    
    current_S = 0
    current_C = 0
    total_pairs = 0
    
    for x in a:
        # 更新当前位置的前缀和 S
        current_S = (current_S + x) % 3
        # 更新当前位置的非3倍数计数 C
        if x % 3 != 0:
            current_C = (current_C + 1) % 3
            
        # 穷举历史前缀可能的 S[j] 值为 s
        for s in range(3):
            # 根据方程：C[j] = (C[r] - (S[r] - S[j])^2) % 3
            diff_S = (current_S - s) % 3
            target_C = (current_C - diff_S * diff_S) % 3
            
            # 累加满足条件的历史前缀数量
            total_pairs += cnt[s][target_C]
            
        # 将当前位置的状态作为历史纪录存入
        cnt[current_S][current_C] += 1
        
    return total_pairs

# 示例测试
# 数组 [3, 6, 9] 全是3的倍数，任意区间的 weight 都是0，全能被3整除
# 长度为 3 的数组共有 3 + 2 + 1 = 6 个子数组
print(solve_weight_divisible_by_3(3, [3, 6, 9]))  # 输出: 6

```

### 复杂度分析

* **时间复杂度**：$O(n)$。我们只需单趟扫描数组，内部的 `for s in range(3)` 循环是常数级别的固定 3 次迭代。
* **空间复杂度**：$O(1)$。我们只开辟了一个 $3 \times 3$ 的固定计数器，不需要存储完整的前缀和数组。即使数据规模达到 $10^6$，也能在毫秒级内跑完。
