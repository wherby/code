
# 判定树上的点集集合在不在一条路径上

https://codeforces.com/gym/105757/problem/N
https://github.com/Yawn-Sean/Daily_CF_Problems/blob/main/daily_problems/2026/06/0611/solution/cf105757n.md


在算法竞赛和图论中，你针对 $k=2$ 时所运用的这套“向上爬升、动态染色并监控度数”的线性算法，在学术和工程上有一个非常响亮且经典的名字：

它属于 **基于虚树（Virtual Tree）或导出子图（Induced Subgraph）的动态拓扑形态判定算法**。更通俗、更具象的称呼是 **“树上点集的路径化判定（Path-like Tree Spanning）”** 或 **“动态树凸包（Tree Convex Hull）度数监控法”**。

这套算法能够成立，其背后的数学与图论依据极其硬核且漂亮。我们可以将其拆解为以下四大核心定理。

---

## 1. 核心图论依据：树上连通子图的“路径判定定理”

这是整个算法最根本的法理依据。
在图论中，如果树 $T$ 的一个顶点子集 $S$ 是**连通**的（在你的代码中，通过“从新点不断往上跳到已访问点”，天然保证了每次加入新点后，已被染色的点集都是绝对连通的），那么：

> **核心定理：一个连通的树节点集合 $S$ 能排成一条“简单路径”，当且仅当 $S$ 的导出子图中，所有点的度数（Degree）最大值 $\le 2$。**

* **依据推导**：
* 在一棵树中，如果一个连通图里没有任何一个点的度数超过 2，那它在拓扑结构上就只能是一条“线”（即由两个度数为 1 的端点，和若干个度数为 2 的中间点组成）。
* 一旦集合中出现任意一个点的度数 $\ge 3$，说明这里分叉了，变成了“星状”或“三叉路口”结构。在树上，任何三叉结构都绝对不可能被一条不重复的简单路径贯穿。
* 你的代码里 `if degs[cur] > 2: flg = False` 正是这一拓扑定理的完美映射。



---

## 2. 数学依据：MEX 的“连续包含性”与“单向阻断性”

为什么我们可以放心地从小到大枚举 $0, 1, 2 \dots$？为什么中途一旦 `flg = False` 就可以直接断定后续的答案？

> **性质：如果一条路径的 $\text{MEX}$ 值为 $M$，那么这条路径必须完整包含整个连续集合 $\{0, 1, 2, \dots, M-1\}$。**

* **依据推导**：
* 如果我们成功把 $0, 1, \dots, i-1$ 都纳入了路径，且它们符合路径形态（最大度数 $\le 2$），那么当前的 $\text{MEX}$ 至少可以达到 $i$。
* 接下来我们尝试把 $i$ 加进来。如果加入 $i$ 导致了度数超标（出现了三叉分叉），说明在当前树的拓扑下，**已经不存在任何一条简单路径能同时包含 $0$ 到 $i$ 了**。
* 既然连 $0 \sim i$ 都包不全，那么未来更广的集合 $0 \sim i+1, 0 \sim i+2$ 必然更不可能包全。因此，分叉具有**单向阻断性**，一旦发生，演进立刻封顶，之前的 $i$ 就是 $k=2$ 时的极限。



---

## 3. 几何依据：树上“唯一起点回溯”的唯一路径性

在传统的算法中，判断点集是否共线通常需要计算复杂的 **LCA（最近公共祖先）**。但你选择将 $0$ 设为全局的根，每次从新点 $i$ 强行单向往上跳。

> **性质：在有根树中，任何一个节点到根节点（0）的简单路径是有且仅有一条的。**

* **依据推导**：
* 既然 $0$ 本身就在路径中，那么任何新加入的点 $i$ 想要与 $0$ 连通，它通往 $0$ 的物理路径是唯一的（就是顺着 `parent` 指针一路上溯）。
* 你的算法相当于在做“虚树的边集动态激活”。每遇到一个新点，就把它到已知路径的这截“唯一盲区”给激活并染色。
* 这种“只往上跳，撞到即停”的操作，不仅完美利用了树的路径唯一性，更在数学上证明了：**你激活的边集和点集，正好是包含 $0 \sim i$ 节点的最小连通子树。**



---

## 4. 算法复杂度依据：摊销时间复杂度（Amortized Analysis）

为什么这个看起来像暴力的 `while not vis[cur]` 循环，整体时间复杂度是纯线性的 $O(n)$，而不是 $O(n^2)$？

> **数学依据：树的边数不变量定理。一棵拥有 $n$ 个顶点的树，有且仅有 $n-1$ 条边。**

* **依据推导**：
* 在你的 `while` 循环中，一个节点只有在 `vis[cur] == 0` 的时候才会被访问，并且在访问的瞬间立刻被置为 `vis[cur] = 1`。
* 每一个节点、每一条边，在整个大循环（$1$ 到 $n-1$）的运行过程中，**有且仅有一次**经历“从 0 变 1”的激活过程。
* 那些已经被激活的点（`vis == 1`），再次被遇到时会触发 `while` 条件失败立刻退出。
* 因此，虽然单次看某个 $i$ 的 `while` 循环可能要跳很多步，但把所有 $i$ 的跳步总量加起来，**刚好等于整棵树的边数 $n-1$**。这就是最经典的**摊销分析**，保证了算法宏观上的线性高效率。



### 总结

你写的这个算法，本质上是在用“导出的动态虚树度数”**去实时监控**“连续 MEX 顶点的共线状态”**。它的名字叫**动态连通凸包度数判定法，其依据是**树的边数有限性**、**根路径唯一性**、以及**简单路径的度数上限为 2** 这三个坚不可摧的图论定理。
