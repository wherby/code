#

https://codeforces.com/gym/106463/problem/A

5 3 0 1
1 2 3

为了理解这个算法的每一步，我们先对输入进行解析：
* **$N=5$**: 棋盘有 5 个格子（0-4）。
* **$K=3$**: 共有 3 次移动。
* **$X=0, Y=1$**: 落在 0 会被传送到 1。
* **$a = [1, 2, 3]$**: 移动序列。

我们需要计算所有初始起点 $s \in \{1, 2, 3, 4\}$ 最终的位置。

---

### 第一步：计算位移前缀和 `nums`
我们不考虑传送门，先看累积的总位移。
* $a_1 = 1 \implies nums[0] = 1$
* $a_2 = 2 \implies nums[1] = (1+2) \pmod 5 = 3$
* $a_3 = 3 \implies nums[2] = (3+3) \pmod 5 = 1$

**结果**：`nums = [1, 3, 1]`

---

### 第二步：后缀预处理 `target` 数组
我们要从后（第 2 步）往前（第 0 步）计算：**“如果我在第 $i$ 步掉进 $X=0$，最终会去哪？”**

#### 1. 处理 $i = 2$ (最后一步):
* 此时位置被传送到 $Y=1$。
* 后面没有移动了。
* **结果**：`target[2] = 1`。
* 更新 `vis`: `vis[nums[2]] = vis[1] = 2`。

#### 2. 处理 $i = 1$:
* 你在第 1 步掉进 $X=0$，被传送到 $Y=1$。还剩第 2 步（位移 $a_3=3$）要走。
* 计算是否会再次撞坑：`to_find_pos = (nums[1] + X - Y) % 5 = (3 + 0 - 1) % 5 = 2`。
* 检查 `vis[2]`：是 `-1`（说明剩下的步数里不会再撞 $X=0$）。
* 计算终点：从 $Y=1$ 走完剩余步数。剩余步数是 $nums[2] - nums[1] = 1 - 3 = -2 \equiv 3 \pmod 5$。
* 终点：$(1 + 3) \pmod 5 = 4$。
* **结果**：`target[1] = 4`。
* 更新 `vis`: `vis[nums[1]] = vis[3] = 1`。

#### 3. 处理 $i = 0$:
* 你在第 0 步掉进 $X=0$，被传送到 $Y=1$。
* 计算是否会再次撞坑：`to_find_pos = (nums[0] + X - Y) % 5 = (1 + 0 - 1) % 5 = 0`。
* 检查 `vis[0]`：是 `-1`（不撞坑）。
* 计算终点：从 $Y=1$ 走完剩余位移 $nums[2]-nums[0] = 1-1 = 0$。
* **结果**：`target[0] = 1`。
* 更新 `vis`: `vis[nums[0]] = vis[1] = 0`（覆盖了之前的记录）。

---

### 第三步：计算每个初始点的答案
我们要找每个点 $s$ **第一次**撞坑的时刻。公式：寻找 $nums[j] \equiv (X - s) \pmod 5$。

| 初始点 $s$ | 寻找位移 $(0-s) \pmod 5$ | `vis` 结果 | 计算逻辑 | 最终位置 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | $(0-1) \equiv 4$ | `vis[4] = -1` | 一辈子没撞坑：$(1 + nums[2]) \pmod 5$ | **2** |
| **2** | $(0-2) \equiv 3$ | **`vis[3] = 1`** | 在第 1 步撞坑，取 `target[1]` | **4** |
| **3** | $(0-3) \equiv 2$ | `vis[2] = -1` | 一辈子没撞坑：$(3 + nums[2]) \pmod 5$ | **4** |
| **4** | $(0-4) \equiv 1$ | **`vis[1] = 0`** | 在第 0 步撞坑，取 `target[0]` | **1** |

---

### 总结验证
我们可以挑一个点手动模拟验证：**初始点 $s=4$**。
1.  **第 0 步 ($a_1=1$)**: $4 + 1 = 5 \equiv 0$。**撞坑！** 传送到 $Y=1$。
2.  **第 1 步 ($a_2=2$)**: $1 + 2 = 3$。没撞坑。
3.  **第 2 步 ($a_3=3$)**: $3 + 3 = 6 \equiv 1$。没撞坑。
* **最终位置：1**。与算法结果一致！



这个算法通过 `vis` 数组，就像是在每个起点面前放了一面镜子，一眼就看穿了它会在哪一站“掉进陷阱”，然后直接通过 `target` 预处理表拿到了终点的“入场券”。
