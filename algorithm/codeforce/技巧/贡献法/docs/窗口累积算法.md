### 一、 核心数学映像：滑动窗口的“两极分化”

当我们把所有长度为 $l$ 的子数组排成一排时，根据定义，每一个子数组都可以表示为两个前缀和的差：

* 第 1 个子数组 $(a_1 \dots a_l)$ ＝ $acc[l] - acc[0]$
* 第 2 个子数组 $(a_2 \dots a_{l+1})$ ＝ $acc[l+1] - acc[1]$
* $\dots$
* 最后一个子数组 $(a_{n-l+1} \dots a_n)$ ＝ $acc[n] - acc[n-l]$

如果我们把这 $n - l + 1$ 个等式**垂直相加**，就会惊奇地发现，所有的 $acc$ 项自动分成了两个互不干扰的**连续大方阵**：

$$\sum \text{所有长度为 } l \text{ 的子数组和} = \underbrace{\Big( acc[l] + acc[l+1] + \dots + acc[n] \Big)}_{\text{正项大方阵 (Positive Group)}} - \underbrace{\Big( acc[0] + acc[1] + \dots + acc[n-l] \Big)}_{\text{负项大方阵 (Negative Group)}}$$

---

### 二、 两种极致的 $O(1)$ 维护策略

正如你的提示所言，要计算这两个连续的大区间和，我们在编写代码时有两种绝妙的 $O(1)$ 路线可选：

#### 路线 A：前缀和的前缀和（空间换时间，逻辑极其对称）

既然 $acc$ 数组是一维前缀和，那么“一维前缀和的区间和”，自然可以用“前缀和的前缀和”（我们不妨记为 `double_acc`）在 $O(1)$ 内直接减出来。

* 正项大方阵的边界是从 $l$ 到 $n$：

$$\text{Positive Sum} = double\_acc[n + 1] - double\_acc[l]$$


* 负项大方阵的边界是从 $0$ 到 $n-l$：

$$\text{Negative Sum} = double\_acc[n - l + 1] - double\_acc[0]$$



#### 路线 B：随着 $l$ 的增长动态指针维护（零额外空间，时间极致）

如果你不想开辟 `double_acc` 数组，我们可以让 $l$ 从 $1$ 逐渐递增到 $n$。当 $l \to l+1$ 时：

* **正项大方阵的变化**：左边界吐出了 $acc[l]$，所以整体减去 $acc[l]$。
* **负项大方阵的变化**：右边界吐出了 $acc[n-l]$，所以整体减去 $acc[n-l]$。

由于每次变化都只涉及到边界上**某一个单元素**的离开，我们只需要用两个变量实时 `-= acc[l]` 和 `-= acc[n-l]`，就能在不借助二次前缀和的情况下，以 $O(1)$ 的开销丝滑地滑过所有长度。

### 三、 完美的算法终稿

基于你的提示，这里给出一个极其清爽、工业级完美的 Python 实现。这里采用**路线 A（前缀和的前缀和）**，它的数学对称美简直让人赏心悦目：

```python
def solve_average_of_averages(n, a):
    # 1. 一阶前缀和 acc (1-indexed 方便处理)
    # acc[i] 表示前 i 个元素的和
    acc = [0] * (n + 1)
    for i in range(n):
        acc[i + 1] = acc[i] + a[i]
        
    # 2. 二阶前缀和 double_acc
    # double_acc[i] 表示 acc[0] 到 acc[i-1] 的总和
    double_acc = [0] * (n + 2)
    for i in range(n + 1):
        double_acc[i + 1] = double_acc[i] + acc[i]
        
    total_avg_sum = 0.0
    
    # 3. 完美对应提示：一起算所有长度为 l 的子数组
    for l in range(1, n + 1):
        # 正项区间和: acc[l] + ... + acc[n]
        pos_sum = double_acc[n + 1] - double_acc[l]
        
        # 负项区间和: acc[0] + ... + acc[n-l]
        neg_sum = double_acc[n - l + 1] - double_acc[0]
        
        # 长度为 l 的所有子数组的元素总和
        sum_of_len_l = pos_sum - neg_sum
        
        # 乘以 1/l，累加到总平均值中
        total_avg_sum += sum_of_len_l / l
        
    # 4. 除以总子数组个数 C(n+1, 2) = n * (n + 1) / 2
    total_subarrays = n * (n + 1) / 2
    return total_avg_sum / total_subarrays

```

### 一、 核心亮点：把区间和变成“贪心滑动叠加”

让我们死死盯住你循环里的核心两行：

```python
cur += acc[n - i] - acc[i]
ans += cur / (i + 1)

```

这里的 `i` 实际上代表的是（子数组长度 $- 1$）。也就是说，当 `i = 0` 时，你算的是长度为 1 的子数组；当 `i = n - 1` 时，你算的是长度为 $n$ 的子数组。

我们来看 `cur` 的递推过程：

* **`i = 0` (长度为 1)：** `cur = (acc[n] - acc[0])`。这其实是**整个数组所有元素的总和**！因为长度为 1 的子数组一共有 $n$ 个（即 $a_0, a_1, \dots, a_{n-1}$），把它们加起来，每个元素恰好出现 1 次，总和刚好是 `acc[n] - acc[0]`。
* **`i = 1` (长度为 2)：** `cur` 在原来的基础上，加上了 `acc[n - 1] - acc[1]`。
此时最新的 `cur = (acc[n] - acc[0]) + (acc[n - 1] - acc[1])`。
稍微展开一下：`acc[n] - acc[1]` 是从第 2 个到最后一个元素的和，`acc[n - 1] - acc[0]` 是从第一个到倒数第二个元素的和。这不正好就是**所有长度为 2 的子数组的和**吗！

你通过 `cur += acc[n - i] - acc[i]`，利用上一次长度为 $l$ 的大方阵结果，**只通过一次前缀和切片剥离**，就丝滑地生出了长度为 $l+1$ 的全新大方阵。这种“叠罗汉”式的动态维护，比开辟二阶前缀和数组更加省内存，运行效率更高！

### 二、 两个需要注意的隐性 Bug（边界与精度）

虽然整体算法框架是完全正确的，但在实际评测（尤其是面对大数或严苛的 Codeforces / LeetCode 评测机）时，有两处细节如果不修正，可能会导致 `WA` (答案错误) 或 `TLE` / `Memory Error`。

#### 1. 精度丢失风险：`ans += cur / (i + 1)`

在 Python 中，`/` 是浮点数除法。你在循环中每一层都除以 `(i + 1)` 并加到 `ans` 里。
如果 $n$ 很大（比如 $n = 2 \times 10^5$），每次循环都进行一次浮点数除法，**浮点数的尾数截断误差会随着 $2 \times 10^5$ 次循环不断累加**，最终导致最后几位小数不准确。

**💡 优化方案**：在循环中只做整数加法，最后出了循环再做除法。
也就是我们可以维护一个数组或者计算一个组合权重，但因为你的 `cur` 包含了除以 `(i + 1)` 的操作，如果想变成纯整数运算，我们可以在最后的 `outs.append` 时统一处理，或者借助分数，不过在这道题中，我们可以**将累加和除法解耦**，或者直接使用 Python 的 `fractions`（但会变慢）。如果评测精度要求是 $10^{-6}$，目前这样也能压线过，但更好的做法是尽可能把除法往后推。

#### 2. 分母的浮点数安全

最后一行的 `ans / (n * (n + 1) / 2)`，其中 `n * (n + 1) / 2` 用了 `/` 会自动转成 float。如果 $n$ 极大，`n * (n + 1)` 的精确整型值在转成 float 时可能会损失低位精度。
**💡 优化方案**：先用地板除 `// 2` 算出总子数组数（保证是完美的整数），再用 `ans / total`。

### 🛠️ 工业级完美的最终提交版

根据你的代码微调后的最稳健版本：

```python
from itertools import accumulate
import sys

# 优化输入
def II(): return int(sys.stdin.readline())
def LII(): return list(map(int, sys.stdin.readline().split()))

def main():
    t = II()
    outs = []
    
    for _ in range(t):
        n = II()
        nums = LII()
        
        ans = 0.0
        acc = list(accumulate(nums, initial=0))
        
        cur = 0
        for i in range(n):
            # 完美的 O(1) 状态剥离动态维护
            cur += acc[n - i] - acc[i]
            # 算入长度为 (i + 1) 的贡献
            ans += cur / (i + 1)
        
        # 稳健的整数地板除，防止大数转浮点时精度溢出
        total_subarrays = n * (n + 1) // 2
        outs.append(ans / total_subarrays)
        
    # 格式化输出，保留 10 位小数确保稳过精度关
    print('\n'.join(f"{x:.10f}" for x in outs))

if __name__ == '__main__':
    main()

```
