# MEX 引入
MEX 可以使得非连续的空间坍缩成连续空间，这样可以更好处理

这是一个非常深刻的数学逻辑问题。之所以这段复杂的 `MEX` 运算最后殊途同归指向了“质因子个数 $\Omega(i)$”，是因为**约数集合的特殊结构**。

我们可以通过 **数学归纳法** 来拆解这个现象：

### 1. 基础情况：$i = 1$
* `sg[1]`：没有比 1 小的正约数，所以 `sg_vis[1]` 为 0。
* `MEX({0})`：最小未出现的非负整数是 0。
* **结果：** `sg[1] = 0`。此时 $\Omega(1) = 0$，成立。

### 2. 质数情况：$i = p$
* $p$ 只有一个正约数：$1$。
* `sg_vis[p]`：会合并 `sg[1]` 的结果，即 `(1 << sg[1])` $\rightarrow$ `(1 << 0)`。
* 集合变为 `{0}`。
* `MEX({0}) = 1`。
* **结果：** `sg[p] = 1`。此时 $\Omega(p) = 1$，成立。

### 3. 递推逻辑：为什么是 $\Omega(i)$？
假设对于所有比 $i$ 小的数，$sg(n) = \Omega(n)$ 都成立。现在看 $i$：

由于题目规定可以移动到**任意**正约数，这意味着从 $i$ 出发，你可以到达所有的约数 $d$。
* 根据假设，这些约数的 SG 值就是他们的质因子个数：$\{ \Omega(d) \mid d \text{ 是 } i \text{ 的约数, } d < i \}$。
* **关键点：** 如果 $i$ 有 $k$ 个质因子（即 $\Omega(i) = k$），那么它的约数 $d$ 的质因子个数 $\Omega(d)$ 可以取遍 $[0, k-1]$ 中的**每一个**整数。
    > *例如：* 如果 $i = 12$ ($2 \times 2 \times 3$), $\Omega(12) = 3$。
    > * 它的约数可以有 0 个质因子（1）
    > * 1 个质因子（2, 3）
    > * 2 个质因子（4, 6）
    > * 这些约数对应的 SG 值集合就是 $\{0, 1, 2\}$。

* **执行 MEX：** $MEX(\{0, 1, 2, \dots, k-1\}) = k$。
* **结论：** $sg(i) = k = \Omega(i)$。

---

### 4. 模版中的位运算在做什么？
回到你的代码段：

```python
while cur > 1:
    # 这里在把所有约数的 SG 值（即它们的质因子个数）打到 bitmask 里
    sg_vis[i] |= sg_vis[i // pr[cur]] | (1 << sg[i // pr[cur]])
    cur //= pr[cur]
```

这行代码实际上在做**并集**。它保证了如果 $i$ 有 $k$ 个质因子，那么 `sg_vis[i]` 的第 $0, 1, 2, \dots, k-1$ 位都会被染成 1。



### 5. `bit_length()` 的物理意义
当 `sg_vis[i]` 的低 $k$ 位全是 1 时（二进制表示为 `...0111...1`，共 $k$ 个 1）：
1.  `~sg_vis[i]`：会将这连续的 $k$ 个 1 变成 0，而原本第 $k$ 位（从 0 开始算）的 0 变成了 1。
    * *例如：* 如果 $k=3$, `sg_vis` 是 `0111` (7)。取反后低位变成 `...1000`。
2.  `x &= -x`：提取出这个最低位的 1（即第 $k$ 位）。
3.  `x.bit_length() - 1`：
    * 如果 $x = 8$ (二进制 `1000`)，`bit_length()` 是 4。
    * $4 - 1 = 3$。
    * **结果正好就是 $k$。**

### 总结
这就是为什么这个复杂的模版最后算出来的就是质因子个数。
* **数学本质：** 约数格（Divisor Lattice）的深度等于质因子总数。
* **代码本质：** 它在用位运算模拟这个深度的搜索和 MEX 判定。

真正能体现 MEX 威力、让 $\Omega(n)$ 彻底失效的，是那些**“打破了质因子线性减少”**的规则。1. 真正的“降维打击”：带有限制的约数博弈想象这样一个规则：你可以移动到 $n$ 的任何约数 $m$，但 $m$ 不能是偶数（除非 $n$ 本身就是奇数）。如果 $n=12$ ($2^2 \times 3^1$)：按照 $\Omega(n)$，它的值应该是 3。但在新规则下，12 的合法后继只有 $\{3, 1\}$（奇数约数）。$SG(1)=0$, $SG(3)=1$。$SG(12) = MEX(\{0, 1\}) = 2$。结论：这里 $SG(12) \neq \Omega(12)$。因为规则“切断”了 12 走向 6 或 4 的路径。$\Omega(n)$ 无法处理这种“奇偶校验”或“条件过滤”，但你的 MEX 模板只需要在 while 循环里加一个 if 判断就能搞定。2. 经典案例：Green Hackenbush 的分叉如果博弈不是在“数”上，而是在“结构”上，$\Omega(n)$ 就完全没用了。看这个结构：一个节点连接了两个分支，左边是长为 2 的竹竿，右边是长为 3 的竹竿。错误直觉：总长度是 $2+3=5$，所以 SG 是 5？错。MEX 逻辑：左分支的 $SG=2$，右分支的 $SG=3$。在这个分叉点，它们的价值先进行 Nim-sum（异或）：$2 \oplus 3 = 1$。这个分叉点相对于地面的 $SG = 1 + 1 = 2$。结论：这种“分型”或“嵌套”的结构，必须通过 MEX 递归地把上层的复杂性“坍缩”成一个简单的数字。3. 为什么你的模板是“终极方案”？你的模板其实是在处理有向无环图 (DAG) 上的博弈。$\Omega(n)$ 只是 DAG 在一种极其特殊、极其完美的结构（格结构）下的快捷键。MEX 模板 是在扫描整张地图。只要题目加上哪怕一点点“不完美”的限制，比如：“不能移动到完全平方数”“只能移动到比当前数小，且相差不超过 100 的约数”“如果剩下的数是质数，下一个玩家必须跳过回合”在这些情况下，质因子个数 $\Omega(n)$ 会瞬间崩塌，而你的 sg_vis 位掩码合并依然稳如泰山。它记录的是**“在这个不完美的世界里，我到底能跳到哪些价值的状态”**。所以，与其说它是为了算质数，不如说它是一个**“博弈地图扫描仪”**，质数只是它在平原地区扫描时顺带算出的结果。
