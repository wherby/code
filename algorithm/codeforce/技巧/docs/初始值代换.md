### 核心解析：如何 $O(n)$ 验证与求解

根据你的提示，当第一个齿轮半径设为未知数 $r$ 时，后续所有齿轮的半径都可以表示为：

* $g_1 = r$
* $g_2 = (x_2 - x_1) - r$
* $g_3 = (x_3 - x_2) - (x_2 - x_1) + r = x_3 - 2x_2 + x_1 + r$
* ……

归纳起来，齿轮半径序列可以写成：


$$g_i = (-1)^{i-1} \cdot r + C_i$$


其中常数项 $C_i$ 可以通过 $O(n)$ 的一次遍历递推出来：$C_1 = 0$，$C_i = (x_i - x_{i-1}) - C_{i-1}$。

#### 第一步：锁定 $r$ 的候选值

因为原有的半径集合 $R$ 全部是正数，且我们知道 $R$ 中的最小值 $R_{min} = \min(r_i)$。
那么实际摆放的齿轮序列 $\text{placement}$ 中的最小值也必然等于 $R_{min}$。
由于 $g_i$ 分为两类：

1. **奇数项：** $r + C_i$ （随 $r$ 增大而增大）
2. **偶数项：** $-r + C_i$ （随 $r$ 减小而增大）

所以，最小值 $R_{min}$ 只能在以下两个位置之一被取得：

* **可能 1：** 某个奇数项达到了最小值。此时 $r = R_{min} - \min(C_{\text{odd}})$
* **可能 2：** 某个偶数项达到了最小值。此时 $r = \max(C_{\text{even}}) - R_{min}$

通过这两条公式，你瞬间就把解的空间缩小到了**最多 2 个候选的 $r$ 值**！

### 第二步：如何 $O(n)$ 判断两个数组元素完全相等？

当我们代入候选的 $r$，算出完整的推导序列 $G = [g_1, g_2, \dots, g_n]$ 后，需要验证 $G$ 是否是原半径集合 $R$ 的一个合法排列。

如果使用排序验证，复杂度是 $O(n \log n)$。若要做到严格的 **$O(n)$**，有以下两种主流做法：

#### 方法 A：哈希计数法（Frequency Map）

利用哈希表（或数组，如果半径数值范围较小）统计元素出现的频次。

1. 遍历原集合 $R$，统计每个半径出现的次数。
2. 遍历推导出的序列 $G$，在哈希表中对其频次进行扣减。
3. 如果扣减过程中出现频次小于 0，或者最终有残余，则说明不匹配。

* **时间复杂度：** $O(n)$
* **空间复杂度：** $O(n)$

#### 方法 B：随机化哈希 / 校验和（仅适用于高要求或特定打卡题）

如果限制空间，也可以采用对数组求和、求平方和，或者利用随机哈希映射求异或和等方式来进行 $O(n)$ 概率验证，但在面试或标准竞赛中，**方法 A（频次统计）** 是最稳妥且标准的 $O(n)$ 线性解法。

### 完整算法流程

1. **递推常数项：** 遍历一次坐标数组 $X$，计算出所有的 $C_i$。在遍历过程中，顺便记录奇数项 $C_i$ 的最小值 $\min(C_{\text{odd}})$，以及偶数项 $C_i$ 的最大值 $\max(C_{\text{even}})$。
2. **提取最小值：** 找到原输入半径数组 $R$ 的最小值 $R_{min}$。
3. **计算候选 $r$：**
* $r_A = R_{min} - \min(C_{\text{odd}})$
* $r_B = \max(C_{\text{even}}) - R_{min}$


4. **线性验证：** 对 $r_A$ 和 $r_B$ 分别生成对应的 $G$ 序列。
* 检查 $G$ 中的所有元素是否都大于 0（半径必须为正）。
* 使用**哈希计数法**在 $O(n)$ 时间内验证 $G$ 和 $R$ 是否包含完全相同的元素。


5. **输出结果：** 只要有一个候选值通过验证，该 $G$ 序列即为正确答案。
